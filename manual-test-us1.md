## Acceptance Scenario 1: 搜尋歌名
**Given** 使用者在首頁  
**When** 在搜尋框輸入「稻香」並按下搜尋  
**Then** 系統顯示所有歌名完全匹配「稻香」的歌曲列表

### 測試步驟
1. [ ] 開啟 http://localhost:5174/
2. [ ] 在搜尋框輸入「稻香」

4. [ ] 驗證：顯示載入指示器
5. [ ] 驗證：搜尋結果顯示歌名包含「稻香」的歌曲
6. [ ] 驗證：每個結果顯示歌名和歌手名稱
7. [ ] 驗證：結果數量 > 0

**結果**: ⏳ 待測試  
**備註**: 需要 Google Sheets 中有「稻香」這首歌

## Acceptance Scenario 4: 修改搜尋條件
**Given** 搜尋結果已顯示  
**When** 使用者修改搜尋框內容並再次搜尋  
**Then** 系統更新搜尋結果

### 測試步驟
1. [ ] 先搜尋一個關鍵字（如「歌」）並記錄結果數量
2. [ ] 修改搜尋框為另一個關鍵字（如「愛」）
3. [ ] 點擊搜尋按鈕
4. [ ] 驗證：結果列表已更新
5. [ ] 驗證：新結果與第一次搜尋的結果不同
6. [ ] 驗證：URL 或狀態已更新

**結果**: ⏳ 待測試  
**備註**: 需要至少 2 組不同的測試資料

## Acceptance Scenario 5: 分頁顯示
**Given** 搜尋返回超過 20 筆結果  
**When** 使用者檢視結果  
**Then** 系統顯示前 20 筆並在底部提供分頁按鈕

### 測試步驟

3. [ ] 驗證：底部顯示分頁控制元件
4. [ ] 驗證：顯示當前頁碼和總頁數
5. [ ] 點擊「下一頁」或頁碼 2
6. [ ] 驗證：結果列表更新為第 2 頁內容
7. [ ] 驗證：URL 或狀態已更新頁碼

**結果**: ⏳ 待測試  
**備註**: 需要 Google Sheets 中有足夠多的資料（> 20 筆相同關鍵字）

## Edge Cases 測試

### 空白輸入
1. [ ] 在搜尋框不輸入任何內容（或只有空格）
2. [ ] 點擊搜尋按鈕
3. [ ] 驗證：顯示提示訊息「請輸入搜尋關鍵字」或類似訊息
4. [ ] 驗證：未執行搜尋 API 請求

**結果**: ⏳ 待測試

### 超長輸入
1. [ ] 輸入超過 200 個字元的字串
2. [ ] 驗證：輸入被限制或顯示警告
3. [ ] 嘗試搜尋（如果允許）
4. [ ] 驗證：系統正確處理

**結果**: ⏳ 待測試

### 特殊字元
1. [ ] 輸入包含特殊字元的搜尋詞（如 `*?%`）
2. [ ] 點擊搜尋
3. [ ] 驗證：系統正確處理（不會崩潰或出錯）

**結果**: ⏳ 待測試

### 防抖機制
1. [ ] 快速連續輸入多個字元
2. [ ] 觀察網路請求（開啟瀏覽器 DevTools Network tab）
3. [ ] 驗證：API 請求在停止輸入後約 400ms 才發送
4. [ ] 驗證：不會為每個字元都發送請求

**結果**: ⏳ 待測試

### 網路錯誤處理
1. [ ] 關閉 Google Apps Script API 或中斷網路
2. [ ] 嘗試搜尋
3. [ ] 驗證：顯示友善錯誤訊息「搜尋失敗，請稍後再試」
4. [ ] 驗證：頁面不會崩潰，保持可互動

**結果**: ⏳ 待測試
