## User Story 1: 基本關鍵字搜尋

### AC1: 搜尋歌名
- [ ] 輸入存在的歌名（例如：「青花瓷」）
- [ ] 驗證顯示符合的歌曲結果
- [ ] 驗證結果包含歌名和歌手名稱
- [ ] 驗證無歌詞片段顯示

### AC2: 搜尋歌手
- [ ] 輸入存在的歌手名稱（例如：「周杰倫」）
- [ ] 驗證顯示該歌手的所有歌曲
- [ ] 驗證結果包含歌名和歌手名稱

### AC3: 搜尋不存在關鍵字
- [ ] 輸入不存在的關鍵字（例如：「xyzabc123」）
- [ ] 驗證顯示「查無結果」訊息
- [ ] 驗證無錯誤訊息

### AC4: 空白輸入驗證
- [ ] 不輸入任何內容直接搜尋
- [ ] 驗證顯示「請輸入搜尋關鍵字」訊息
- [ ] 輸入僅包含空白字元
- [ ] 驗證顯示相同錯誤訊息

### AC5: 超長輸入處理
- [ ] 輸入超過 200 字元的關鍵字
- [ ] 驗證顯示「搜尋關鍵字不可超過 200 字元」訊息

### AC6: 修改搜尋條件
- [ ] 先搜尋「青花瓷」
- [ ] 觀察結果數量
- [ ] 修改為「周杰倫」
- [ ] 驗證結果更新且數量不同

### AC7: 分頁顯示
- [ ] 搜尋一個常見字（例如：「的」）確保超過 20 筆結果
- [ ] 驗證顯示分頁元件
- [ ] 驗證顯示「第 1 頁，共 X 頁」
- [ ] 點擊「下一頁」
- [ ] 驗證顯示第 2 頁結果
- [ ] 點擊「上一頁」
- [ ] 驗證返回第 1 頁

### AC8: 防抖機制
- [ ] 快速連續輸入多個字元
- [ ] 觀察是否等待 400ms 後才執行搜尋
- [ ] 驗證不會每次輸入都觸發搜尋

## User Story 2: 歌詞片段搜尋與高亮

### AC1: 歌詞搜尋與片段顯示
- [ ] 輸入存在於歌詞中的文字（例如：「天青色等煙雨」）
- [ ] 驗證顯示包含該歌詞的歌曲
- [ ] 驗證顯示歌詞片段（3 行）
- [ ] 驗證匹配文字以黃色背景+粗體顯示

### AC2: 歌詞片段固定 3 行
- [ ] 搜尋歌詞中間的文字
- [ ] 驗證顯示匹配行 + 前 1 行 + 後 1 行
- [ ] 搜尋歌詞開頭的文字
- [ ] 驗證顯示匹配行 + 後 2 行（或實際可用行數）
- [ ] 搜尋歌詞結尾的文字
- [ ] 驗證顯示匹配行 + 前 2 行（或實際可用行數）

### AC3: 多處匹配顯示第一處
- [ ] 搜尋在歌詞中多次出現的文字（例如：「你」）
- [ ] 驗證僅顯示第一處匹配的片段
- [ ] 驗證高亮第一處出現的文字

### AC4: 特殊字元處理
- [ ] 搜尋包含特殊字元的歌詞（例如：「愛（Love）」）
- [ ] 驗證正確匹配和高亮
- [ ] 驗證無 JavaScript 錯誤

### AC5: 歌詞搜尋結果分頁
- [ ] 搜尋常見歌詞文字確保超過 20 筆結果
- [ ] 驗證顯示分頁元件
- [ ] 驗證每頁顯示 20 筆結果
- [ ] 切換頁面驗證歌詞片段正確顯示

## User Story 3: 點擊進入歌曲詳細頁面

### AC1: 點擊導航
- [ ] 在搜尋結果頁面點擊任一歌曲
- [ ] 驗證瀏覽器導航到 `/songs/:id`
- [ ] 驗證 URL 包含歌曲 ID

### AC2: 詳細頁面內容
- [ ] 驗證顯示完整歌名
- [ ] 驗證顯示完整歌手名稱
- [ ] 驗證顯示完整歌詞（保留換行格式）
- [ ] 驗證顯示「返回搜尋結果」按鈕

### AC3: 返回保持狀態
- [ ] 搜尋關鍵字（例如：「周杰倫」）
- [ ] 點擊任一結果進入詳細頁面
- [ ] 點擊「返回搜尋結果」按鈕
- [ ] 驗證返回搜尋結果頁面
- [ ] 驗證搜尋關鍵字仍然保留
- [ ] 驗證搜尋結果仍然顯示

### AC4: Hover 效果
- [ ] 將滑鼠移到搜尋結果項目上
- [ ] 驗證顯示 hover 效果（背景色變化、陰影等）
- [ ] 驗證滑鼠指標變為 pointer
- [ ] 移開滑鼠驗證效果消失

### AC5: 歌曲不存在處理
- [ ] 直接在瀏覽器輸入不存在的歌曲 ID（例如：`/songs/nonexistent`）
- [ ] 驗證顯示友善錯誤訊息
- [ ] 驗證顯示返回首頁連結
